# 三、研究目标与研究内容

本课题着眼于开发一个基于微服务架构的智能人才招聘管理平台，实现招聘全流程的数字化管理，并集成AI智能评估能力。该系统包括人才管理、职位管理、简历管理、面试管理、消息通知等核心功能模块，同时提供企业管理后台和求职者门户双端界面。

## 3.1 研究目标

1. **架构目标**：设计并实现基于微服务的系统架构，将系统拆分为6个独立的微服务，确保系统高可用、易扩展、便于维护。

2. **功能目标**：实现招聘全流程管理，覆盖人才库管理、职位发布、简历筛选、面试安排、消息通知等核心业务场景。

3. **智能化目标**：集成Coze工作流AI能力，实现简历智能评估和人岗匹配推荐，提升招聘效率和匹配精度。

4. **体验目标**：采用现代化前端技术，提供美观易用的用户界面，支持企业端和求职者端双端使用。

## 3.2 研究内容

本系统主要包括后端微服务和前端界面两大部分：

### 一、后端微服务功能

**1. 用户服务（User Service）**
- 用户注册、登录、注销等账号管理功能
- 基于JWT的身份认证和权限控制
- 用户信息管理和个人资料维护
- 角色权限管理（管理员、HR、面试官、求职者）

**2. 人才服务（Talent Service）**
- 人才信息的增删改查操作
- 人才库搜索和筛选功能
- 人才标签管理和分类
- 人才状态跟踪（待联系、面试中、已录用等）

**3. 职位服务（Job Service）**
- 职位的发布、编辑、删除操作
- 职位状态管理（招聘中、已暂停、已关闭）
- 职位搜索和筛选功能
- 职位统计和数据分析

**4. 简历服务（Resume Service）**
- 简历上传和解析功能
- 简历状态流转管理
- 求职申请记录管理
- AI智能评估接口集成

**5. 面试服务（Interview Service）**
- 面试安排和日程管理
- 面试官分配和日程查询
- 面试反馈记录和评分
- 面试统计和数据分析

**6. 消息服务（Message Service）**
- 站内消息发送和接收
- 系统通知和提醒
- 消息已读状态管理
- 未读消息统计

### 二、前端界面功能

**1. 企业管理后台**
- 仪表板：数据概览、趋势图表、快捷操作
- 人才管理：人才列表、详情查看、状态管理
- 职位管理：职位发布、编辑、状态控制
- 简历管理：简历列表、状态流转、AI评估
- 面试日历：面试安排、日程管理、反馈记录
- 招聘看板：可视化招聘流程管理
- 消息中心：站内消息、系统通知
- 数据报表：招聘数据统计和分析
- 系统设置：权限管理、操作日志

**2. 求职者门户**
- 首页：平台介绍、热门职位推荐
- 职位列表：职位搜索、筛选、详情查看
- 企业招聘：企业信息展示、在招职位
- 我的投递：申请记录、状态跟踪
- 我的简历：简历编辑和管理

### 三、系统重点内容

**1. 微服务架构设计**

系统采用微服务架构，将业务功能拆分为6个独立的服务，每个服务独立部署、独立数据库，通过RESTful API进行服务间通信。这种架构具有以下优势：
- 服务独立部署，故障隔离
- 技术栈灵活，可针对不同服务选择最适合的技术
- 易于水平扩展，应对高并发场景
- 便于团队分工协作

**2. AI智能评估集成**

系统集成Coze工作流平台，实现简历智能评估功能：
- 简历内容智能解析和结构化
- 基于职位要求的匹配度评分
- 候选人优劣势分析
- 面试建议生成

通过AI评估，可以大幅减少HR的简历筛选工作量，提高人岗匹配的准确性。

**3. 前端用户体验**

采用Vue3 + TypeScript + Element Plus技术栈，实现现代化的用户界面：
- 响应式设计，支持多端访问
- 数据可视化，直观展示招聘数据
- 交互友好，操作流程简洁
- 实时更新，及时反馈操作结果
