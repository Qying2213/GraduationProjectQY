# 五、研究工作进度安排

| 周次 | 时间阶段 | 主要任务 |
|------|----------|----------|
| 第1周 | 选题阶段 | 确定论文主题方向，进行论文题目的筛选，明确研究方向为基于微服务架构的智能人才招聘管理平台。 |
| 第2周 | 调研阶段 | 以论文题目为核心，对相关资料进行收集和翻阅，调研国内外招聘管理系统的发展现状和技术方案。 |
| 第3周 | 开题阶段 | 对已搜集的资料加以整理，论证分析论文的可行性、实际性，将论文题目和大致范围确定下来，完成开题报告。 |
| 第4-5周 | 设计阶段 | 整合已有资料，明确系统需求，确定技术栈（Go + Vue3 + PostgreSQL），完成系统架构设计、数据库设计、API接口设计，搭建项目框架与代码版本管理。 |
| 第6-7周 | 后端开发（一） | 完成用户服务（user-service）和职位服务（job-service）的开发，实现用户注册登录、权限管理、职位CRUD等功能。 |
| 第8-9周 | 后端开发（二） | 完成人才服务（talent-service）和简历服务（resume-service）的开发，实现人才库管理、简历上传解析、申请记录管理等功能。 |
| 第10周 | 后端开发（三） | 完成面试服务（interview-service）和消息服务（message-service）的开发，实现面试安排、反馈记录、站内消息等功能。 |
| 第11-12周 | 前端开发 | 完成企业管理后台和求职者门户的前端界面开发，包括仪表板、人才管理、职位管理、简历管理、面试日历、消息中心等页面。 |
| 第13周 | AI集成 | 集成Coze工作流平台，实现简历智能评估功能，完成AI评估接口对接和结果展示。 |
| 第14周 | 测试优化 | 进行系统功能测试、接口测试，修复发现的Bug，优化系统性能和用户体验。准备论文中期检查。 |
| 第15周 | 论文初稿 | 完成论文的初稿撰写，整理系统设计文档、技术实现细节、测试结果等内容。 |
| 第16周 | 论文修改 | 向指导老师寻求意见，优化论文的结构，润色语句，修改不当之处，补充不足之处。 |
| 第17周 | 论文定稿 | 论文资料整合，最终定稿，准备答辩材料，制作答辩PPT。 |
| 第18周 | 论文答辩 | 熟悉论文内容，增强对论文内容的把握，进行论文答辩。 |

## 阶段性成果

| 阶段 | 预期成果 |
|------|----------|
| 设计阶段 | 系统架构设计文档、数据库设计文档、API接口文档 |
| 开发阶段 | 6个后端微服务、前端管理后台、求职者门户 |
| 测试阶段 | 测试报告、Bug修复记录 |
| 论文阶段 | 毕业论文、答辩PPT |
