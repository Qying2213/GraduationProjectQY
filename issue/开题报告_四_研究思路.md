# 四、拟采用的研究思路（方法、技术路线、可行性论证等）

## 二、框架设计

### 1. 微服务架构设计

系统将业务功能拆分为6个独立的微服务，每个服务职责单一、边界清晰：

| 服务名称 | 端口 | 职责描述 |
|---------|------|----------|
| user-service | 8081 | 用户认证、权限管理 |
| job-service | 8082 | 职位管理、职位统计 |
| interview-service | 8083 | 面试安排、反馈管理 |
| resume-service | 8084 | 简历管理、AI评估集成 |
| message-service | 8085 | 消息通知、站内信 |
| talent-service | 8086 | 人才库管理、搜索 |

每个服务独立运行，通过RESTful API进行通信，服务之间松耦合，便于独立开发和部署。

### 2. 数据库设计

系统采用PostgreSQL关系型数据库，主要数据表包括：

- **users**：用户信息表，存储账号、密码、角色等
- **jobs**：职位信息表，存储职位名称、要求、薪资等
- **talents**：人才信息表，存储候选人基本信息、技能等
- **resumes**：简历表，存储简历内容、状态、评估结果等
- **applications**：申请记录表，关联职位和人才
- **interviews**：面试安排表，存储面试时间、地点、面试官等
- **interview_feedbacks**：面试反馈表，存储评分和评价
- **messages**：消息表，存储站内消息和通知

### 3. AI智能评估设计

系统通过集成Coze工作流平台实现AI智能评估功能：

```
简历上传 → 内容提取 → Coze API调用 → 评估结果解析 → 结果存储
                            │
                    ┌───────┴───────┐
                    │  Coze工作流    │
                    │  - 简历解析    │
                    │  - 技能匹配    │
                    │  - 综合评分    │
                    │  - 建议生成    │
                    └───────────────┘
```

评估结果包括：匹配度评分、技能分析、优劣势总结、面试建议等。

### 4. 前端架构设计

前端采用Vue3组合式API + TypeScript，主要模块包括：

- **路由管理**：Vue Router实现页面路由和权限控制
- **状态管理**：Pinia实现全局状态管理
- **API封装**：Axios封装HTTP请求，统一错误处理
- **UI组件**：Element Plus提供丰富的UI组件库

## 四、可行性论证

### 1. 技术可行性

- **后端技术**：Go语言性能优异，Gin框架轻量高效，适合微服务开发，相关技术文档和社区资源丰富。
- **前端技术**：Vue3生态成熟，Element Plus组件丰富，TypeScript提供类型安全，开发效率高。
- **数据库**：PostgreSQL功能强大，支持JSON数据类型，适合存储结构化和半结构化数据。
- **AI服务**：Coze平台提供成熟的工作流能力，API调用简单，支持自定义工作流。

### 2. 资源可行性

- **开发环境**：macOS系统，VS Code/Kiro IDE开发工具
- **运行环境**：本地开发测试，支持Docker容器化部署
- **学习资源**：丰富的官方文档、开源项目和技术社区

### 3. 时间可行性

项目开发周期约16周，各阶段任务明确，时间安排合理，具备按期完成的条件。
